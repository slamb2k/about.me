# Component Forge UI Library

Welcome to the Component Forge UI component library! This is a collection of reusable, accessible, and well-tested React components built with TypeScript and Tailwind CSS.

## Getting Started

This library provides a comprehensive set of UI components designed for modern web applications. All components are:

- **Accessible**: Built with WCAG 2.1 AA compliance in mind
- **Responsive**: Mobile-first design that works across all devices
- **Type-safe**: Full TypeScript support with comprehensive type definitions
- **Animated**: Smooth animations using Framer Motion
- **Tested**: Comprehensive test coverage with automated accessibility testing

## Component Categories

### Core UI Components

- **Button**: Versatile button component with multiple variants and sizes
- **Card**: Flexible card layout with header, content, and footer sections
- **Section**: Layout component for consistent page section structure
- **AnimatedText**: Text with smooth fade-in animations
- **IconButton**: Specialized button for icon-only interactions

### Layout Components

- **LazySection**: Performance-optimized section with intersection observer
- **OptimizedImage**: Image component with lazy loading and responsive design

### Utility Components

- **SkipNavigation**: Accessibility component for keyboard navigation
- **VirtualList**: Performance component for large lists

## Design System

The components follow a consistent design system based on:

- **Colors**: Tailwind CSS color palette with semantic naming
- **Typography**: Responsive typography scale
- **Spacing**: Consistent spacing using Tailwind's spacing scale
- **Shadows**: Subtle shadows for depth and hierarchy
- **Border Radius**: Consistent border radius for modern appearance

## Accessibility Features

All components include:

- Proper ARIA labels and descriptions
- Keyboard navigation support
- Screen reader compatibility
- High contrast support
- Focus management

## Animation Philosophy

Animations are designed to:

- Enhance user experience without being distracting
- Provide meaningful feedback for user interactions
- Respect user preferences for reduced motion
- Maintain performance across devices

## Development Guidelines

When using these components:

1. Always provide proper `aria-label` attributes for interactive elements
2. Use semantic HTML structure
3. Test with keyboard navigation
4. Verify screen reader compatibility
5. Consider mobile and touch interactions

## Browser Support

Components are tested and supported in:

- Chrome (latest 2 versions)
- Firefox (latest 2 versions)
- Safari (latest 2 versions)
- Edge (latest 2 versions)
